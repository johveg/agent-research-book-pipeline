# Weekly Curation Instruction

Daily runs may collect, classify, extract, and propose.

Weekly curation decides what should change in the book.

Do not allow daily noise to rewrite the book every day.

## Weekly curation goals

The weekly curation process should:

1. Review the week’s new sources.
2. Review new candidate entities.
3. Review new candidate claims.
4. Review trend candidates.
5. Identify strong evidence.
6. Identify weak or noisy signals.
7. Identify contradictions.
8. Decide whether chapters should be updated.
9. Decide whether new chapter briefs are needed.
10. Decide whether old claims should be revised or demoted.

## Weekly review questions

Ask:

- What did we learn this week?
- What changed from the previous understanding?
- Which claims became stronger?
- Which claims became weaker?
- Which trends are real enough to monitor?
- Which trends should be rejected?
- Which sources are best?
- Which sources are weak?
- Which chapter sections need updates?
- Which sections should not change?
- Are there contradictions or corrections?
- Did any privacy/publication risks appear?

## Weekly output

Produce a weekly editorial report with:

1. Executive summary.
2. Source quality summary.
3. New entities worth keeping.
4. Candidate claims promoted.
5. Candidate claims rejected.
6. Weak signals to monitor.
7. Trends accepted/rejected.
8. Contradictions or corrections.
9. Recommended chapter updates.
10. Chapters not ready for update.
11. Required human decisions.
12. Next week’s watchlist.

## Chapter update policy

Update chapters weekly only if:

- new supported claims exist,
- old claims need correction,
- structure needs improvement,
- a meaningful contradiction appeared,
- or the Editor approves a material improvement.

Do not update chapters just because new material was captured.

## Final rule

Daily is for collection and preparation.

Weekly is for judgment and book movement.
