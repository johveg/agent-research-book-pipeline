# Chapter Brief Instruction

Every chapter must have a chapter brief before the Author writes or revises it.

The chapter brief defines the purpose, argument, boundaries, evidence requirements, and publication constraints.

The Author must follow the brief.

The Editor must review against the brief.

## Required chapter brief fields

Each brief must include:

1. Chapter title.
2. Purpose.
3. Target reader.
4. Main argument.
5. Required concepts.
6. Required claims.
7. Required examples.
8. Allowed source types.
9. Excluded source types.
10. Open questions.
11. What the chapter must not claim.
12. Desired tone.
13. Desired length.
14. Related entities.
15. Publication readiness criteria.

## Chapter brief template

Use this template:

```markdown
# Chapter Brief: <chapter title>

## Purpose

Explain what this chapter is meant to achieve.

## Target reader

Describe who the chapter is for.

## Main argument

State the central argument in one or two paragraphs.

## Required concepts

List concepts that must be explained.

## Required claims

List claim IDs or claim types that must be used.

## Required examples

List examples that should be included if supported.

## Allowed source types

Define which source types may support the chapter.

## Excluded source types

Define which sources must not be used or must be caveated.

## Open questions

List unresolved questions that should remain visible.

## What this chapter must not claim

List overstatements, unsupported conclusions, or risky claims to avoid.

## Desired tone

Define the tone for this chapter.

## Desired length

Define approximate length or section depth.

## Related entities

List relevant entities.

## Publication readiness criteria

Define what must be true before this chapter can be published.
```
