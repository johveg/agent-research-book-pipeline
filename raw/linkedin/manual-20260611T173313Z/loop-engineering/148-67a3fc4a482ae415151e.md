## Visible text

Anthropic and OpenAI engineers are both telling software engineers to write loops. That is not a coincidence. For the last two years, most of us have used AI like a smart intern. We give it one prompt, read the answer, fix what is wrong, ask again, and keep repeating that cycle ourselves. The model does one step. We do the rest. We remember the goal, decide the next move, judge quality, and stop only when the work feels good enough. In that setup, the human is still the loop. 1. A loop simply means you move that repetition into the system. Instead of prompting the agent again and again by hand, you define a cycle once: Understand the goal, plan the next step, do the work, check the result, fix what failed, and repeat until a clear condition is met. “Write code, run tests, read errors, fix, retest until green” is a loop. “Draft content, score it against a checklist, rewrite until it passes” is a loop. A prompt asks for output. A loop keeps pushing toward an outcome. 2. The bottleneck is no longer typing prompts. The actual bottleneck is building a process the model can follow without drifting, forgetting, or producing polished garbage. That is why strong engineers now care about memory files, evals, subagents, worktrees, and tool access. These are not fancy extras. They are the rails that keep the loop honest. Without them, the agent burns tokens and wanders. With them, the agent can actually compound work. 3. Most people should not start with giant autonomous swarms. Start with closed loops on repeatable work you already understand. Fix failing tests until all pass. Review pull requests against a checklist. Summarize support tickets and route low-confidence cases to a human. The point is to automate a feedback cycle that already makes sense to you. That is the shift that everyone's been talking about. Prompt engineering helped people get better answers. Loop engineering helps people build systems that keep producing verified results after they press go. --- P.S: This is pretty new and I'm still testing, would love to know if you have insights to offer.
