If we now have loop engineering, then what’s next? Environment engineering. Prompt engineering, context engineering, harness engineering, loop engineering... all of these are useful, but they all assume you have a place to run the agent that reflects the environment a human expert works in. The benchmarks we use today will die or need to change cause that’s not where experts actually work. Experts work inside messy, annoying, frustratingly specific environments. Different tools, different data, different permissions, different edge cases, and every company has its own version of “yeah this is just how we do it here.” If your agent isn’t trained and tested inside that, you’re not really testing whether it can do the job reliably and accurately. If you haven’t already read the viral article by Sarah Guo, she talks about how the bottleneck for AI is not intelligence, but the ability to get it into these very places: ”Transforming a firm isn’t a single task you can write an eval for. It takes an operator to moneyball it, with extremely ambiguous intermediate goals and incomplete feedback, over very long horizons, in an environment that won’t hold still.” This is the part I think people are underestimating. Rubrics matter. Verifiers matter. Loops matter. But rubrics and verifiers only provide value if the environment is realistic enough to expose the failures that will go into improving your agent. And its very hard to define these environments & conditions, let alone have access to them or build them (thats a whole other problem I’ll save for another post). It’s unsexy, but I think it’s where most of the value is going to come from. My bet: whoever ends up with the best environments ends up with the best agents (the new AI FDEs already know this wink). John DiFederico , Anthony Jirouschek, Rich Alldrin & I run into new weird edge cases every day that we literally have never thought of, and this is a team that has built, deployed & touched every SIEM & detection product under the sun. You’ve already seen me reference Vivek Trivedy several times, so I don’t have to preach this is where the industry is going!
