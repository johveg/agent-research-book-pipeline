Feed post Michael Moor • 3rd+ Assistant Professor for Medical AI at ETH Zurich, D-BSSE. Core Faculty ETH AI Center. Previously: Stanford Computer Science. 2h • Follow Running AI agents has become easy. Observing them is the hard part. Let one run 24/7 for days and your terminal becomes a wall of scrollback no human can ever read. We don't know what it actually did, where it stalled, or whether it quietly went off the rails. A long-horizon agent feels like a black box: by the time it finishes (or breaks), the signal is buried under thousands of lines of tool calls, stack traces, and intermediate results. In an attempt to address this "human out-of-the-loop" problem, we built a human-friendly interface for agents beyond clutterly log files. Meet AgentGUI - a pixel-art office for observing and steering your agents, currently built for Hermes agents. Every task is an agent sitting at a desk. The floor is your whole fleet at a glance. Click a desk to drop into what that agent is doing, right now. Quick clarification, because "agents in a 2D world" sounds familiar. This is *not* another agent-society sim (AI Hospital, Generative Agents, etc.) where agents "live," gossip, and interact with each other. That's not the point. AgentGUI isn't a simulation for agents - it's an agent observability interface for humans. The pixel world exists purely for **you**, the human, to observe and audit what real working agents have actually been doing, since last time you checked in. Open any desk and you get every view a run needs: • Activity feed: chat history, tool calls, results • Run breakdown: reasoning / generation / tools / idle over the whole run • Debug terminal: live sandbox shell I/O, Agent console • File tree: artifacts it made and more.. Observing isn't enough, you need to steer. From the UI you can interrupt, reassign, or redirect any agent mid-task. Customize profiles, drag and drop them. Click a desk, hand an agent new work. No SSH, no killing processes. And for jobs too long to babysit: a agent manager patrols the floor. It audits each agent's work against its (editable) `TASK.md`, marks it solved - or steers it back on course if the output drifts. Long-running tasks get guided and auto-resumed. Everything is reviewable after the fact. Save a trajectory and load it later to replay exactly what an agent did, step by step; for review, debugging, or audit. The run doesn't vanish when the terminal closes. Built local-first. Agents run in Docker sandboxes, inference can be 100% local (Ollama / vLLM) or API-based (for now via Hermes), and you can easily configure profiles and tool sets per agent before or at runtime. Your traces never have to leave your machine. The app, the agent, everything can be easily locally hosted. Agents are going to run longer and more autonomously. The bottleneck won't be capability, it'll be observability and control. AgentGUI aims to tackle exactly that. This was the result of a fun lab hackathon last week.
