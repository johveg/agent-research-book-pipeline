The model does something magical. Then it hits production. And slowly, things break. The agent forgets context. Gets stuck in loops. Ignores instructions. The team huddles and someone says the words spoken in a hundred war rooms before them: “We need a better model.” They upgrade. It breaks again. Here’s are the five more recent disciplines that separate teams who figure it out from teams who keep upgrading models forever. Prompt Engineering is everyone’s first move. Personas, few-shot examples, chain-of-thought nudges. For a single turn, it’s a superpower. But it’s the art of the opening line — it sets the tone, it cannot carry the conversation. The moment a system needs to stay coherent across thousands of interactions, prompts alone start cracking. Context Engineering is the upgrade. Karpathy defined it best: “the delicate art of filling the context window with just the right information for the next step.” Not what you say — what surrounds the model. What gets retrieved, in what order, how compressed, what gets cut. Research shows accuracy drops sharply when key information is buried mid-context. Prompt engineering is actually a subset of this. Memory Engineering solves the wall every agent hits: sessions end, models forget. The fix isn’t just a vector database — it’s four layers. Working memory (live context), episodic memory (timestamped events), semantic memory (distilled patterns), procedural memory (reusable skills). The hard part is consolidation — knowing when to compress raw episodes into durable knowledge without losing what made them distinct. Harness Engineering is the infrastructure exoskeleton the model never sees but entirely depends on — session management, tool dispatch, guardrails, observability, context routing. If the session layer drops state between turns, the agent gets amnesia and doesn’t know it. One 2025 finding shows improving the harness on the same model can outperform switching to a better model entirely. The brain was never the bottleneck. The nervous system was. Loop Engineering governs the think → act → observe → repeat cycle. Infinite spin. Context anxiety — the agent quietly abandons long tasks when its context fills up. Silent failures mid-loop with no recovery. Loop engineering makes the cycle bounded, observable, and recoverable. Think of it as flight envelope protection — invisible until it’s the only thing that mattered. Most broken agents aren’t broken models.
