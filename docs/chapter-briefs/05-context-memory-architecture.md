# Chapter Brief: 5. Context and Memory Architecture

## Purpose

Explain how context and memory shape agent behavior, continuity, and risk.

## Target reader

Intelligent operators, architects, engineers, builders, and technical leaders trying to understand emerging agent systems without hype.

## Main argument

The chapter should distinguish what an agent sees now from what a system stores and recalls later. It should describe context/memory as architecture decisions with tradeoffs.

## Required concepts

- context window
- working context
- persistent memory
- retrieval
- skills
- state
- privacy

## Required claims

Use definition, technical pattern, risk, limitation, and example claims about context and memory architecture.

## Required examples

- A memory preference
- a retrieval corpus
- a skill as procedural memory, if supported.

## Allowed source types

A/B sources may support factual claims. C sources may support interpretation if caveated. D social/search material may only support weak signals or examples when caveated. E sources must not support chapter facts.

## Excluded source types

Raw LinkedIn/web captures, private/session-visible material, credentials, cookies, logs, E-quality sources, and unsupported repeated social posts. D sources require caveats and cannot establish strong facts.

## Desired tone
